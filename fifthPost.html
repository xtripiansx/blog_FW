Update

november 30, 2020 - december 6, 2020

This week I tried different ways to connect my azure devops pipeline with the jenkins pipeline. I first had a service hook, that was triggered when there was a push to the source code. It would make a connection to the jenkins server and run a selected build.

It worked but I was disappointed that there was no visual cues of what was triggered and when it was done. I also did not know what happened in Jenkins. After following some tutorials I decided to make a build pipeline. In the azure-pipeline.yml file I created a Task.

The task made a connection to the jenkins server and ran the given build. That approach satisfied me, because I saw if failed or passed based on the logs that Jenkins sent to azure devops.

It took me a while to figure out how to make the pipeline, because I first thought to build the application before, but it was not necessary. Next step was to deploy the application to a Kubernetes cluster, so I began to read some docs.
